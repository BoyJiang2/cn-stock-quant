# 赚钱优先路线图：数据、策略、消息面与 Agent 协作

更新日期：2026-06-19

> 2026-06-20 因子实验室、指数隔离、AI 受控候选协议和本轮真实实验结果见：
> `docs/factor-lab-and-ai-research-handoff.md`。

本文是后续 agent 的主交接文档。目标不是先做概念展示，而是把 `cn-stock-quant` 逐步做成一个能持续发现、验证、淘汰和执行 A 股策略的本地量化系统。

## 当前状态

已具备：

- 股票代码规范化：支持 `000001`、`SZ000001`、`000001.SZ` 等输入统一入库和查询。
- 数据中心诊断：股票列表、单票状态、研究池同步进度、批量同步。
- 研究股票池回测：回测请求可使用 `symbol_source="research_pool"` 自动选取本地完整覆盖股票。
- 低波抗跌策略：`low_vol_defensive` 已包含趋势、波动率、成交额、回撤、相对基准下行贝塔等因子。
- 回测策略上下文：`StrategyContext.benchmark_history` 已可传入策略。
- 指数同步入口：`/api/data/sync/index` 已支持沪深300、中证500、中证1000、创业板指，带 AkShare 重试。
- 前端回测页：支持手动股票/研究池、基准选择、策略参数、风控参数。

当前限制：

- 本地完整样本仍太小，最近验证只有 28 只股票完整覆盖。
- 交易日历未接入，非交易日起点会影响研究池覆盖判断。
- 指数仍临时写入 `daily_bars`，后续应一等化，避免指数和股票代码命名空间混淆。
- 历史回测详情尚未完整持久化 `symbol_source`、`selected_symbols`、`benchmark_curve`。
- 尚无消息面、公告、舆情、研报数据。

## 总目标

系统最终应回答三个问题：

1. 今天市场适合进攻、防守、空仓还是轮动？
2. 哪些策略/因子最近真正有效，哪些已经退化？
3. 明天具体应该买什么、卖什么、买多少，最大风险是多少？

所有 AI、消息面、因子和策略扩展，都必须服务于这三个问题。

## Phase 1：稳定当前底座

优先级最高。

任务：

- 接入交易日历，研究池覆盖判断改为交易日覆盖，而不是自然日覆盖。
- 将指数数据一等化：
  - 方案 A：新增 `index_daily_bars` 表。
  - 方案 B：给 `daily_bars` 增加 `asset_type`，取值 `stock/index/etf`。
  - 短期推荐方案 A，迁移风险小，语义清晰。
- 回测结果持久化增强：
  - 保存 `symbol_source`
  - 保存 `selected_symbols`
  - 保存 `benchmark_symbol`
  - 保存 `benchmark_curve`
- 优化研究池同步：
  - 失败重试
  - 跳过空数据
  - 继续下一批
  - 前端显示失败原因
- 做 300 股票 synthetic 性能测试，防止后面样本扩大后回测变慢。

验收：

- `2024-01-01` 这类非交易日起点不会误判研究池无覆盖。
- 研究池回测能稳定返回选中股票列表和完整回测结果。
- 后端完整测试通过。
- 前端 build 通过。

目标：先扩到 300-800 只可回测股票，策略比较才有意义。

任务：

## Phase 3：赚钱导向策略池

目标：不要押单一策略，而是建立候选策略池，让系统自己比较谁有效。

优先新增策略：

1. 低波红利策略
   - 低波动
   - 高股息
   - 低回撤
   - 成交额过滤
   - 适合震荡和弱市

2. 小盘动量策略
   - 20/60/120 日动量
   - 成交额放大
   - 过滤一字板和过热涨停
   - 适合牛市和结构行情

3. ETF 轮动策略
   - 沪深300、中证500、中证1000、创业板、红利、证券、半导体、新能源、医药等 ETF
   - 优先用于模拟盘，因为 ETF 退市和财务造假风险低

4. 均值反转策略
   - 短期超跌
   - 长期趋势不坏
   - 成交额未崩
   - 适合震荡市

5. 波动收缩突破策略
   - 长期横盘
   - 波动率压缩
   - 放量突破
   - 适合阶段主升行情

6. 质量价值策略
   - PE/PB/ROE/现金流/负债率/股息率
   - 需要财务数据源后实现

每个策略必须输出 `dict[symbol -> weight]`，不得绕过现有目标仓位接口。

验收：

- 每个策略都有参数元信息。
- 每个策略都有最小测试。
- 每个策略可以在研究池上统一回测。
- 回测报告能显示年化、回撤、Sharpe、Calmar、月度胜率、超额收益。

## Phase 4：消息面与市场情绪引擎

这是新增重点。消息面不是“聊天总结”，而是要变成可回测、可比较、可风控的结构化数据和因子。

### 4.1 数据源分层

第一批低成本数据源：

- AkShare 新闻接口
- 东方财富财经新闻
- 东方财富股吧
- 巨潮资讯公告
- 雪球讨论
- 财联社/证券时报等公开新闻源

第二批增强数据源：

- 研报标题和摘要
- 百度指数/微信指数/微博热搜等热度数据
- 交易所问询函、监管公告
- 龙虎榜、北向资金、融资融券

注意：

- 先做公开、稳定、低成本源。
- 对抓取频率做限制，避免封禁。
- 所有原文要保存来源、URL、发布时间、抓取时间。

### 4.2 后端数据模型

建议新增表：

- `news_items`
  - `id`
  - `source`
  - `url`
  - `title`
  - `content`
  - `published_at`
  - `fetched_at`
  - `symbols`
  - `market_tags`
  - `dedupe_hash`

- `sentiment_scores`
  - `id`
  - `news_id`
  - `symbol`
  - `trade_date`
  - `sentiment`
  - `confidence`
  - `event_type`
  - `impact_horizon`
  - `model_name`

- `market_sentiment_daily`
  - `trade_date`
  - `market_score`
  - `panic_score`
  - `euphoria_score`
  - `policy_score`
  - `hot_topics`

### 4.3 情绪因子

第一批可回测因子：

- 单股新闻情绪均值：最近 1/3/5/20 日新闻情绪。
- 单股新闻热度：新闻数量、讨论数量、环比变化。
- 负面冲击因子：高置信负面新闻数量。
- 公告风险因子：减持、监管、问询、诉讼、业绩预亏。
- 政策利好因子：行业政策、补贴、产业规划。
- 市场恐慌因子：全市场负面新闻占比、股吧恐慌词频。
- 过热因子：讨论暴增、情绪极端乐观、成交额异常放大。

情绪因子使用原则：

- 情绪不是直接买卖信号，先作为过滤器和仓位调节器。
- 强负面新闻优先进入风控，而不是追求收益。
- 极端乐观通常用于降权或止盈，不用于无脑追涨。

### 4.4 LLM 分析层

LLM 用途：

- 新闻去重和归因。
- 把新闻分类为政策、业绩、监管、行业、舆情、突发事件。
- 判断影响方向：利好、利空、中性、不确定。
- 给出影响周期：盘中、1-3 天、1-4 周、长期。
- 生成每日市场摘要。
- 给策略回测报告写中文解释。

推荐接口边界：

- `NewsProvider`：只负责抓新闻。
- `SentimentAnalyzer`：只负责把文本转成结构化评分。
- `SentimentFactorEngine`：只负责按交易日聚合成因子。
- `Strategy`：只能读取聚合后的因子，不直接抓网页或调用 LLM。

### 4.5 前端页面

新增“消息面/情绪中心”：

- 今日市场情绪仪表盘
- 全市场恐慌/过热指数
- 热点主题列表
- 单股消息流
- 单股情绪曲线
- 负面预警列表
- 持仓相关新闻

回测页增强：

- 策略可选择是否启用情绪过滤。
- 展示“启用情绪前 vs 启用情绪后”的收益/回撤差异。

### 4.6 交易辅助

模拟盘阶段要接：

- 每日收盘后扫描持仓新闻。
- 如果持仓出现高置信负面事件，降低目标仓位或提示人工确认。
- 如果全市场恐慌指数极高，降低进攻策略权重。
- 如果行业政策利好集中出现，只进入候选池，不直接满仓。

验收：

- 能抓取并入库至少一个新闻源。
- 能按股票和日期查询相关新闻。
- 能生成每日市场情绪分数。
- 至少一个策略可使用情绪过滤做 A/B 回测。

## Phase 5：因子实验室

目标：批量测试因子，不靠手感选策略。

因子类别：

- 动量：20/60/120 日收益、新高距离。
- 反转：5 日跌幅、20 日均线偏离、RSI。
- 波动：20 日波动率、下行波动率、最大回撤、ATR。
- 流动性：成交额、量比、成交额稳定性。
- 基本面：PE、PB、ROE、股息率、现金流。
- 消息面：情绪、热度、负面冲击、政策利好、过热。

报告：

- IC
- Rank IC
- 分组收益
- 多空收益
- 换手率
- 衰减曲线
- 因子相关性

## Phase 6：策略排行榜与元策略

目标：让系统自动判断当前该用哪些策略。

策略排行榜指标：

- 年化收益
- 最大回撤
- Sharpe
- Calmar
- 超额收益
- 月度胜率
- 最近 3/6/12 月表现
- 情绪冲击下表现
- 手续费压力测试表现

元策略：

- 牛市：小盘动量、突破、行业轮动。
- 熊市：低波红利、ETF 防守、现金。
- 震荡：均值反转、红利、低波。
- 高恐慌：降仓、减少小盘暴露。
- 极端乐观：降低追涨策略权重。

## Phase 7：模拟盘闭环

目标：从回测变成每日交易辅助。

每日流程：

1. 同步行情。
2. 同步新闻/公告/舆情。
3. 计算行情因子和情绪因子。
4. 跑策略池。
5. 元策略分配资金。
6. 风控裁剪目标仓位。
7. 生成次日交易计划。
8. 前端展示买入、卖出、原因、风险。

## Phase 8：三 Agent 协作方式

按用户要求，后续复杂开发采用三方均衡协作。

推荐分工：

- Codex：协调者、接口设计、回测引擎、测试、集成、最终验证。
- GLM：数据源、因子实验室、前端数据中心、中文报告。
- DeepSeek：策略生成、策略优化、情绪策略、验证建议。

标准流程：

1. 三方分别理解需求并提出方案。
2. Codex 汇总比较三套方案，拆分任务。
3. GLM 负责模块 A。
4. Codex 负责模块 B。
5. DeepSeek 负责模块 C。
6. GLM 阅读 Codex 接口。
7. Codex 阅读 DeepSeek 接口。
8. DeepSeek 阅读 GLM 接口。
9. 三方共同修复集成问题。
10. 运行完整测试、前端构建、真实样本回测。

共享接口文件必须单 owner 合并，避免并发覆盖：

- `backend/app/strategy/base.py`
- `backend/app/schemas/backtest.py`
- `backend/app/data/repository.py`
- `backend/app/backtest/engine.py`
